# Medi Point - Smart Healthcare Appointment Platform

Medi Point is a modern, responsive, and secure healthcare appointment scheduling platform. It allows patients to search for doctors, book appointments, link Google Calendar for automatic syncs, log symptoms for AI-powered extraction, and receive email reminders. The platform is backed by **FastAPI** (Python), **Express.js/EJS** (Node.js), and **Supabase PostgreSQL**.

## 🛠️ Local Setup Guide

Follow these steps to run both the backend and frontend servers locally on your machine.

### Prerequisites
* Python 3.12+
* Node.js v18+
* PostgreSQL or SQLite
* Redis (running locally or in the cloud)

### Step 1: Clone the Repository
```bash
git clone https://github.com/AtharvaDubey25/Medi-Point.git
cd Medi-Point
```

### Step 2: Configure Environment Variables
Copy `.env.example` in the root folder to `.env` (and place a copy in both the `/backend` and `/frontend` folders if needed):
```bash
cp .env.example .env
```
Fill in the database credentials, SendGrid API Key, Groq API Key, and Google Calendar OAuth keys inside `.env`.

### Step 3: Run the Backend (FastAPI)
1. Navigate to the backend directory:
   ```bash
   cd backend
   ```
2. Create and activate a Python virtual environment:
   ```bash
   python -m venv .venv
   # Windows:
   .venv\Scripts\activate
   # Linux/macOS:
   source .venv/bin/activate
   ```
3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
4. Push database schema to Supabase/PostgreSQL:
   ```bash
   python -m prisma db push
   ```
5. Generate the Prisma Client and fetch the binary engines:
   ```bash
   python -m prisma generate
   python -m prisma py fetch
   ```
6. Start the development server:
   ```bash
   python -m uvicorn app.main:app --reload --port 8000
   ```

### Step 4: Run the Frontend (Express.js)
1. Open a new terminal window and navigate to the frontend directory:
   ```bash
   cd frontend
   ```
2. Install Node.js dependencies:
   ```bash
   npm install
   ```
3. Start the Express server:
   ```bash
   node server.js
   ```
4. Open your browser and navigate to `http://localhost:3000`.

## 🗄️ Database Schema (Prisma)

The application uses the following relational database schema:

* **User**: Represents all users of the system (Patients, Doctors, Admins). Stores auth credentials, contact details, and Google Calendar tokens.
* **DoctorProfile**: Contains doctor-specific details such as specialization, bio, consultation fees, and experience. Linked 1-to-1 with a User record.
* **AvailabilitySlot**: Stores availability times (start time, end time, date) generated by doctors.
* **Appointment**: Tracks bookings between patients and doctors, including state (PENDING, CONFIRMED, COMPLETED, CANCELLED).
* **SymptomLog**: Stores patient-submitted symptom notes and the AI-extracted structured information.
* **MedicationReminder**: Stores medication plans and timing alerts for patients.
* **AuditLog**: Stores security and administrative operations for system-wide auditing.

## 🔌 API Documentation

### Authentication (`/api/auth`)
* `POST /api/auth/register` - Create a new user account (Patient/Doctor).
* `POST /api/auth/login` - Verify password and return JWT access token.
* `GET /api/auth/google/callback` - Callback for Google OAuth calendar link.

### Doctors (`/api/doctors`)
* `GET /api/doctors/search` - Search doctors by name or specialization.
* `GET /api/doctors/{id}/slots` - Fetch availability slots for a specific date.

### Appointments (`/api/appointments`)
* `POST /api/appointments/book` - Reserve an availability slot.
* `GET /api/appointments/my` - Fetch current user's appointments.
* `POST /api/appointments/{id}/cancel` - Cancel a scheduled appointment.

### Symptoms & Summaries (`/api/symptoms`)
* `POST /api/symptoms/appointment/{id}` - Log patient symptoms and analyze them via LLM.
* `GET /api/symptoms/appointment/{id}/post-visit-summary` - Generate and fetch AI post-visit summaries.

## 🤖 LLM Prompts (Groq Llama 3.1)

### 1. Symptom Extraction Prompt
```text
Analyze the following patient symptom description: "{text}"
Extract the following information:
1. Symptoms: List of symptoms.
2. Severity: Mild, Moderate, or Severe.
3. Duration: If mentioned.
4. Suggested Specialization: Recommend the type of doctor.
Respond ONLY in this JSON format:
{{
  "symptoms": ["symptom1", "symptom2"],
  "severity": "Mild/Moderate/Severe",
  "duration": "Duration description or null",
  "suggested_specialization": "Specialization"
}}
```

### 2. Post-Visit Summary Prompt
```text
Create a clean, empathetic, and professional Post-Visit Summary for this patient:
Patient Name: {patient_name}
Doctor Name: {doctor_name}
Symptoms Logged: {symptoms}
Doctor's Notes: {notes}

Format your output in Markdown with the following sections:
- **Summary**: Concise recap of the visit and diagnosis.
- **Prescribed Medications**: Clear bulleted list showing dosage and frequency.
- **Lifestyle Recommendations**: Actionable recovery steps.
- **Follow-up Guidelines**: When to seek further care.
```

## 📅 Google Calendar Setup Steps

To set up Google Calendar synchronization, follow these steps:

### 1. Google Cloud Project & Consent
1. Open the [Google Cloud Console](https://console.cloud.google.com/).
2. Create a project and select **Google Auth Platform** on the left menu.
3. In **Branding**, set up your app name, support email, and URLs:
   * **Homepage**: `https://medi-point-portal.onrender.com`
   * **Privacy Policy**: `https://medi-point-portal.onrender.com/privacy`
   * **Terms of Service**: `https://medi-point-portal.onrender.com/terms`
4. Go to the **Audience** tab:
   * Under **Publishing Status**, click **PUBLISH APP** to make the auth screen accessible to external users.
   * Under **Test Users**, click **Add Users** and add your testing email addresses (e.g., `atharvart25@gmail.com`). *Google restricts access to developer-approved test emails until the app is fully verified.*

### 2. Configure Client Credentials
1. Go to the **Clients** tab in Google Auth Platform.
2. Click **Create Client** and select **Web Application**.
3. Configure the following redirect parameters:
   * **Authorized JavaScript origins**:
     * `https://medi-point-portal.onrender.com`
     * `http://localhost:3000` *(for local testing)*
   * **Authorized redirect URIs**:
     * `https://medi-point.onrender.com/api/auth/google/callback`
     * `http://localhost:8000/api/auth/google/callback` *(for local testing)*
4. Save to retrieve your **Client ID** and **Client Secret**. Paste these credentials into your environment variables file.
